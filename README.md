## Project Overview

This project involves building an end-to-end Business Intelligence (BI) solution for AdventureWorks, a global manufacturing company. The project covers the full data lifecycle: from data ingestion and transformation in Power Query to relational modeling, DAX measure development, and the creation of an interactive three-page dashboard designed for executive decision-making.

## Problem Statement

AdventureWorks handles massive volumes of transactional data across multiple years (2015-2017) and regions. However, the company struggled with:

- Data Silos: Sales, products, and customer data were stored in separate files.

- Lack of Profit Visibility: The raw data only contained quantities, making it difficult to calculate actual margins.

- Growth Tracking: Difficulty in comparing year-over-year (YoY) performance to identify if the business was truly expanding.

## Dataset Description

The analysis was performed on a multi-table relational dataset consisting of:

- Fact_Sales: Transactional records (Order dates, keys, and quantities).

- Fact_Returns: Records of returned products.

- Dimension Tables: Detailed metadata for Products, Customers, Territories, and a specialized Calendar table.

## Data Architecture (The "Star Schema")

The project utilizes a Star Schema architecture to optimize query performance. A central Fact table (Sales) is connected to five Dimension tables (Product, Customer, Territory, Calendar, and Returns) via one-to-many relationships. This design minimizes data redundancy and enables efficient cross-filtering across the entire dashboard."

## Tools Used

- Power BI Desktop: For data modeling and visualization.

- Power Query (M): For ETL (Extract, Transform, Load) processes.

- DAX (Data Analysis Expressions): For advanced analytical calculations.

- Star Schema Architecture: For optimized database performance.

## Steps Followed

1. Data Ingestion: Imported raw CSV/Excel files into Power BI.

2. ETL & Transformation: Cleaned data, handled null values, appended yearly sales files, and merged product categories.

3. Data Modeling: Established a Star Schema with one-to-many ($1:*$) relationships and single cross-filter directions.

4. DAX Development: Created calculated columns using the RELATED function and 8+ measures for financial analysis.

5. Dashboard Design: Built a three-page interactive report (Executive, Tactical, and Strategic).

6. Analytical Insight: Interpreted visuals to provide data-driven business recommendations.


## Dashboard Features

- Page 1 (Executive Summary): High-level KPI cards, revenue trends, and geographic map analysis.

- Page 2 (Detailed Analysis): Interactive Decomposition Tree, matrix with conditional formatting, and category drill-downs.

- Page 3 (Insights & Monitoring): Anomaly Detection, Top/Bottom performer analysis, and Year-over-Year growth tracking.

## Key DAX Measures

- Total Revenue: SUM(Fact_Sales[SalesAmount])

- Total Profit: [Total Revenue] - [Total Cost]

- Profit Margin %: DIVIDE([Total Profit], [Total Revenue], 0)

- YoY Growth %: Uses CALCULATE and SAMEPERIODLASTYEAR to measure annual expansion.

- Revenue YTD: TOTALYTD function for cumulative annual tracking.

## Key Insights

- Product Dependency: 85% of profits are generated by the "Bikes" category, indicating a need for diversification.

- Regional Growth: The European market is the fastest-growing region with a 15% YoY increase.

- Return Risks: A 12% spike in returns follows major promotional periods in the Clothing category.

## Challenges Encountered

- Data Normalization: The Fact_Sales table lacked price data. This was resolved by using the RELATED function to fetch unit prices from the Product Dimension.

- Model Performance: Loading multiple years of data created overhead. This was optimized by "Disabling Load" for redundant source queries and implementing a clean Star Schema.

## Conclusion

The AdventureWorks Business Intelligence solution successfully transforms fragmented raw data into a strategic asset. By implementing automated DAX measures and AI-driven visuals like the Decomposition Tree, the dashboard empowers stakeholders to move beyond descriptive reporting and perform diagnostic analysis that protects profit margins and identifies new market opportunities.
